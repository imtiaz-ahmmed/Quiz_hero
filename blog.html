Difference Between Map, filter, forEach and find

map: The map method returns a new array that is the result of applying a function to each element of an existing array. It creates a new array with the same length as the original array, where each element is the result of the function applied to the corresponding element in the original array.
filter: The filter method returns a new array containing all elements of an existing array that pass a test specified by a function. It creates a new array with only the elements that satisfy the specified condition.
forEach: The forEach method iterates over an array and executes a callback function on each element of the array. It does not return a new array; instead, it performs an action on each element of the array.
find: The find method returns the first element of an array that satisfies a specified condition. It returns undefined if no element satisfies the condition.
In summary, map and filter create new arrays, while forEach does not. find returns a single element of an array that satisfies a condition, while map and filter can return multiple elements.

Difference Between Local Storage, Section Storage and Cookies

Local storage, session storage, and cookies are all mechanisms for storing data in the browser, but they differ in their purpose and behavior.
Cookies: Cookies are small text files that are stored on the client-side (browser) and are used to store user-specific information like user preferences, login status, and shopping cart information. Cookies can be set to expire after a specified time or when the user closes the browser.
Local Storage: Local storage is a web storage API that allows data to be stored in the browser beyond the lifetime of a session (even after the browser is closed). Local storage is intended to store large amounts of data that needs to be accessed frequently by the web application. It is more secure than cookies because it is not sent to the server with every request, and it can only be accessed by the web application that created it.
Session Storage: Session storage is similar to local storage in that it is also a web storage API, but it is intended to store data for a single session only. Session storage is deleted when the user closes the browser or the tab. It is useful for storing temporary data that is needed for a specific session, like shopping cart information.
In summary, cookies are used for small amounts of data that need to be sent to the server with every request, local storage is used for large amounts of data that need to be accessed frequently by the web application, and session storage is used for temporary data that is needed for a single session.

What is Event Loop & how it works?

The event loop is a fundamental concept in JavaScript that allows for the execution of code in a non-blocking manner. When JavaScript code is executed, it is executed on a single thread. This means that only one piece of code can be executed at a time, and if a piece of code is blocking (e.g. waiting for a resource to become available), the entire application will be blocked until that resource is available. This can cause the application to become unresponsive, which is a bad user experience. The event loop solves this problem by allowing code to be executed in a non-blocking manner. When an event (such as a user click or a network request) occurs, it is added to a queue. The event loop continuously checks the queue for any new events, and when it finds one, it executes the associated callback function. Because the event loop executes code in a non-blocking manner, the application can continue to respond to other events while waiting for the completion of the current event. To summarize, the event loop is a mechanism that allows JavaScript code to be executed in a non-blocking manner by using a queue to manage events and their associated callback functions. This allows for a more responsive and performant application.
